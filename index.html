Rope as a Text Buffer

A Rope is an ideal data structure for text editors. Unlike a plain array, it supports O(log n) insertions and deletions anywhere in the text, making it perfect for large documents. Below you can see the current string stored in the rope:

Any Insert / Delete operations performed in the Simulator tab also update the rope displayed here. Use the Simulator to explore the splay-tree internals.

Why a Splay Tree?

Splay trees are self-adjusting binary search trees. Every accessed node is rotated ("splayed") to the root, so frequently-accessed nodes are found quickly on subsequent operations. This gives amortised O(log n) time for all operations while naturally keeping the tree reasonably balanced.

Complexity

index(i) → O(log n) amortised (splay to root) insert(text, pos) → O(k log n) where k = len(text) delete(from, to) → O(log n) concat(A, B) → O(log n) split(rope, pos) → O(log n)

About this project

This is a Rope (splay-tree) algorithm visualiser built with JavaScript, HTML and CSS. It is designed to help you understand how the Rope data structure works through interactive step-by-step simulations and visual tree representations.
